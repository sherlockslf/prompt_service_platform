# 提示词管理服务模块设计文档（与当前代码对齐版）

## 1. 模块定位

提示词管理服务不再采用“硬编码双智能体提示词”的Demo模式，当前已收敛为**PSU（Prompt Service Unit）全生命周期管理**：

- 以PSU为最小资产单元管理提示词与结构约束
- 提示词以片段化方式存储于数据库并支持权限分级编辑
- Schema与参数集按PSU采用覆盖写语义维护当前生效值
- 通过编排草稿、版本审核、发布与回滚形成闭环

## 2. 核心能力

### 2.0 生命周期状态约束

- 全流程统一状态：`DRAFT -> CANDIDATE -> FORMAL -> ARCHIVED`
- `DRAFT`：可编辑（Prompt/Schema/参数集）
- `CANDIDATE`：只读待验收
- `FORMAL`：只读且可对外服务，不允许编辑与删除
- `ARCHIVED`：归档，不可对外服务

### 2.1 提示词片段管理

- 数据表：`ai_prompt_prompt_fragments`
- 核心字段：`fragment_key`、`content`、`editable`、`type`、`sort_order`
- 规则：
  - `CORE_RULES`用于锁定核心规则
  - `MESSAGE_TEMPLATE`用于业务话术编排
  - 仅`editable=true`片段允许业务侧更新
  - 定稿后片段会被锁定

### 2.2 Schema覆盖写管理

- 数据表：`ai_prompt_json_schemas`
- 语义：每个PSU仅保留一条当前生效Schema（`psu_id`唯一）
- 兼容字段：`version`保留但固定语义（默认1）
- 变更信息：通过`change_log`与`updated_at`追踪

### 2.3 参数集覆盖写管理

- 数据表：`ai_prompt_param_sets`
- 语义：每个PSU仅保留一条当前生效参数集（`psu_id`唯一）
- 用途：
  - 作为在线调试与审核预览的默认输入基线
  - 配合测试数据集进行覆盖渲染

### 2.4 版本审核与发布联动

- PSU与审核主线统一单字段版本号：`version_no`
- 审核表：`ai_prompt_version_reviews`，唯一约束`(psu_id, version_no)`
- 审核预览：支持按“参数集 + 编排快照”渲染Prompt并返回缺失变量

## 3. API设计（当前实现）

### 3.1 Prompt管理 `/api/prompts/**`

- `GET /api/prompts/{psuId}`：获取片段列表
- `POST /api/prompts`：新增片段
- `PUT /api/prompts/{fragmentId}`：更新片段
- `DELETE /api/prompts/{fragmentId}`：删除片段
- `POST /api/prompts/{fragmentId}/finalize`：定稿片段
- `POST /api/prompts/{psuId}/test`：在线测试Prompt

### 3.2 Schema管理 `/api/schemas/**`

- `GET /api/schemas/{psuId}`：获取当前Schema
- `PUT /api/schemas/{psuId}`：覆盖更新Schema
- `GET /api/schemas/{psuId}/versions`：兼容接口（覆盖写模式下返回当前记录）

### 3.3 参数集管理 `/api/param-sets/**`

- `GET /api/param-sets/{psuId}`：获取当前参数集
- `PUT /api/param-sets/{psuId}`：覆盖更新参数集

### 3.4 编排与审核 `/api/compositions/**` + `/api/versions/**`

- `PUT /api/compositions`：保存编排草稿
- `POST /api/compositions/render`：按输入参数渲染编排
- `POST /api/compositions/submit`：提交编排版本
- `POST /api/versions/{psuId}/submit`：提交审核
- `GET /api/versions/{reviewId}/preview`：审核预览（参数集渲染）
- `POST /api/versions/{reviewId}/review`：审核通过/驳回
- `GET /api/versions/{psuId}/compare?fromVersionNo=&toVersionNo=`：版本对比
- `POST /api/versions/{psuId}/rollback`：版本回滚

## 4. 关键时序

1. 研发创建PSU并维护Schema、参数集
2. 业务侧编排并调试可编辑Prompt片段
3. 业务侧提交版本进入审核
4. 研发在线预览并审核
5. 审核通过后进入发布流程并更新生效指针

## 5. 非目标声明
